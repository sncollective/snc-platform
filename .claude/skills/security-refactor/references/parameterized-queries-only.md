# Rule: Parameterized Queries Only

> All DB queries via Drizzle ORM; no raw SQL string interpolation.

**Domain**: code

## Motivation

OWASP A03 (Injection). Drizzle ORM generates parameterized SQL automatically, making SQL injection structurally impossible when used correctly. This rule ensures no one bypasses the ORM with raw string queries. Already the established pattern — codified to prevent regression.

## Before / After

### From this codebase: existing pattern (correct)

**Before:** *(what a violation would look like)*
```typescript
const content = await sql`
  SELECT * FROM content WHERE id = '${id}' AND deleted_at IS NULL
`; // String interpolation — SQL injection vector
```

**After:** *(the established pattern)*
```typescript
const [content] = await db
  .select()
  .from(contentTable)
  .where(and(eq(contentTable.id, id), isNull(contentTable.deletedAt)));
```

### Synthetic example: dynamic filtering

**Before:**
```typescript
const results = await sql.unsafe(
  `SELECT * FROM users WHERE role = '${role}'`
); // sql.unsafe bypasses parameterization
```

**After:**
```typescript
const results = await db
  .select()
  .from(users)
  .where(eq(users.role, role));
```

## Exceptions

- Drizzle migrations generated by `drizzle-kit` (these contain raw SQL by design)
- Custom SQL via `db:generate --custom` for data backfills (reviewed manually)

## Scope

- Applies to: all application code in `apps/api/src/`
- Does NOT apply to: migration files in `drizzle/`, seed scripts
