# Session: 0.3.0 review sprint — 4 features + 1 story cleared ahead of Animal Future

Reviewed and landed five items against release 0.3.0 (Animal Future live event 2026-04-24, three days out). All items bound at review-pass. Started the session with a dashboard snapshot to pick event-enabling work; ended with the submodule + parent commits (platform@d980d67, root@d69aafb) and four new backlog items captured for 0.3.1 follow-on.

## Arc

1. **[live-page-controls-hover](../active/features/live-page-controls-hover.md)** — kicked back to drafting on first review when acceptance testing surfaced missing `.controlVisible` CSS + a cascade bug in `__root.module.css` + a DOM-region bug (hover handlers on a route-scoped div couldn't cover the GlobalPlayer rendered at root layout). Also surfaced a larger research-alignment gap: [live-streaming-ux-patterns.md](../research/live-streaming-ux-patterns.md) landed 2026-04-20 (two days after the feature's scope) and commits to a broader layout-modes model (theater = near-full-width + dark bg, fullscreen + PiP as peer modes) this feature didn't adopt. Parked [live-page-layout-chrome-decision](../backlog/live-page-layout-chrome-decision.md) as a scoping spike (site-width vs full-bleed /live chrome), narrowed the redesign to bug-fix scope, re-designed, re-implemented, and iterated through review with fix-in-flight for the DOM-region bug (window-level listeners) and several successive polish passes (unified chat toggle, theater-mode top anchoring, overlay pointer-events clearing so the `×` is click-through).

2. **[chat-presence](../active/features/chat-presence.md)** — review surfaced viewer-count decrement lag + cross-tab inconsistency. Partial defensive fixes landed (stale-ref `pruneStaleClients` in `chat-rooms.ts`, roomId filter on `SET_PRESENCE` / `USER_JOINED` / `USER_LEFT` reducer actions) but didn't resolve the observed lag. Root cause is more likely a missing WS heartbeat / server idle-timeout — parked [chat-presence-reliability](../backlog/chat-presence-reliability.md) for 0.3.1. Separately: "can't tab to Show users" turned out to be focus-indicator clipping against `.panel`'s `overflow: hidden`, fixed with an inset outline; polish-tuning of that outline parked as [chat-user-list-toggle-focus-indicator-tuning](../backlog/chat-user-list-toggle-focus-indicator-tuning.md).

3. **[admin-creators-data-table](../active/features/admin-creators-data-table.md)** — two CLS findings in one review pass (sort-arrow reflow on column change, mobile text-wrap splitting arrow from header text). Fixed in-flight with a reserved `.sortIndicator` slot + `white-space: nowrap` on `th`.

4. **[playout-liq-256k-audio-bump](../active/stories/playout-liq-256k-audio-bump.md)** — small one-bitrate bump (AAC 128k → 256k). Carried at `release_binding: 0.3.0` from scope time. Bumped the generator template in `liquidsoap-config.ts` (not just the gitignored static `playout.liq`) so a future `regenerateAndRestart` call produces a matching file. `snc-liquidsoap` restarted healthy; passed with no findings.

5. **[upload-edit-ux-overhaul](../active/features/upload-edit-ux-overhaul.md)** — largest item of the session. 4 implementation units, orchestrated across 3 parallel Sonnet agents after an Explore grounding pass (file-layout verification). Review surfaced two issues: (a) "Processing media…" stuck until manual refresh — pre-existing client-side refresh gap surfaced by the clearer new state machine; parked [content-processing-state-auto-refresh](../backlog/content-processing-state-auto-refresh.md). (b) Unit 4 regression — design mental-model error: the GlobalPlayer's expanded mode renders as a sibling of `<Outlet>` at root layout, not inside VideoDetailView's DOM, so the always-rendered `.videoArea` wrapper left an empty 16:9 black box beside the actual player. Fixed in-flight by wrapping `.videoArea + .playOverlay` in `{!playerExpanded && …}`. S3 cascade-delete verification lifted to the 0.3.0 bundle's `## Prod verification` as prod-only residual.

## Learnings

- **Research docs landing after a feature's scope are a first-class review-blocker.** The `live-page-controls-hover` kick-back to drafting was specifically triggered by [live-streaming-ux-patterns.md](../research/live-streaming-ux-patterns.md) committing to a layout-modes model the feature didn't adopt. The right move wasn't to retrofit the research — it was to park a scoping spike ([live-page-layout-chrome-decision](../backlog/live-page-layout-chrome-decision.md)) that subsumes the broader rework and narrow the feature to bug-fix scope. Both sides win: the event-critical fixes ship, the research-aligned rework gets a fair design pass later.

- **Agent mental models of DOM placement aren't reliable from design text alone.** Unit 4 of upload-edit-ux-overhaul assumed `GlobalPlayer` expanded renders "into the same space via portal-like behavior." It doesn't — it's a root-layout sibling. The design-time fix (`.videoArea` always-rendered wrapper) produced an empty black box at review. The corrective pattern: when a feature crosses component boundaries (this one spans VideoDetailView + GlobalPlayer's root-level expansion), either verify the DOM relationship in the design pass or make the fix conditional on the observed integration.

- **Defensive fixes don't always solve the observed symptom, but they still earn their place.** The `chat-presence` stale-ref cleanup + roomId reducer filter were correct fixes for latent bugs (cascade over-counts, cross-room state contamination) even though they didn't resolve the specific lag the user reported. Shipped them and parked the remaining heartbeat work separately rather than reverting — the partial fixes are still real improvements.

- **Commit scoping via `git commit -- <pathspec>`.** Pre-existing staged WIP (a `tech-reference-lint` backlog → epic promotion rename that wasn't part of this session) was preserved by using pathspec-limited commit rather than the default "commit everything staged" path. Kept that WIP intact for separate handling.
