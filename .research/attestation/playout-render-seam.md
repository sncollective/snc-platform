## Summary

The current playout.liq render seam — `liquidsoap-render.ts` (renders the .liq) +
`playout-topology.ts` (assembles topology from DB channel rows). Read 2026-06-16 to characterize
the *current* channel architecture, which is the baseline several backlog items propose to
change. Confirms the per-channel-static topology the `single-queue-per-output` and
`snctv-fallback-dynamic-channel` items target.

## Key passages

**Per-channel block (`liquidsoap-render.ts` lines 18–63):** each playout channel renders its own
`request.queue(id="channel-<uuid>")`, a `fallback(track_sensitive=false, [<queue>,
mksafe(blank())])` source, its own `output.url(...)` push to SRS, and three harbor.http
endpoints (queue/skip/now-playing). This is the "each channel needs its own `request.queue` +
harbor endpoints baked into `playout.liq`" state the `single-queue-per-output` item describes.

**S/NC TV broadcast block (lines 154–156):**
```
snc_tv = fallback(track_sensitive=false,
  transitions=[notify_switch("live"), notify_switch("queue"), notify_switch("fallback"), notify_switch("blank")],
  [live_source, snc_tv_queue, ${t.broadcast.fallbackSourceVar}, mksafe(blank())])
```
The broadcast fallback chain is `[live_source, snc_tv_queue, <fallbackSourceVar>, blank()]`.

**Static fallback selection (`playout-topology.ts` lines 127, 147–153):**
`const defaultPlayout = channels[0];` → `fallbackSourceVar: defaultPlayout ?
\`${defaultPlayout.liqVar}_source\` : "mksafe(blank())"`. The S/NC TV fallback is the FIRST
playout channel's source, "static selection for now" (comment line 109–110) — exactly the
hardcoded coupling `snctv-fallback-dynamic-channel` wants replaced by a DB-driven
`defaultPlayoutChannelId`.

**Regeneration model (`liquidsoap-render.ts` lines 78–79):** "Auto-generated by S/NC Platform —
regenerated on channel create/delete." Channel CRUD today = regenerate the whole .liq. The
topology is `readonly`/pure (line 112 "Pure — no DB, no filesystem"); render is byte-identical
for identical topologies (line 70).

## Structural metadata

- Current model: one `request.queue` + one `fallback` + one `output.url` per channel; broadcast
  is a 4-tier `fallback`; fallback target is statically `channels[0]`.
- Channel CRUD path today: regenerate .liq, restart (config-generation Option A).
- The render seam is the seam the editorial-engine epic's chosen mechanisms render into.
