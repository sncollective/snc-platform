# Step 1: Golden characterization tests for the current generator

Pin the **current** `generateLiquidsoapConfig` output byte-for-byte before anything moves. These goldens are the verification spine for the whole feature — every later step must keep them green unchanged.

**Files:** `apps/api/tests/services/liquidsoap-config.test.ts` (extend), new snapshot files under `apps/api/tests/services/__snapshots__/`.

**Current state:** tests assert `toContain(...)` fragments — they'd pass even if surrounding text drifted.

**Target state:** add a `describe("golden output")` block using Vitest `await expect(config).toMatchFileSnapshot("./__snapshots__/<case>.liq")` for four canonical inputs:

1. zero playout channels (pins the doubled `mksafe(blank())` fallback)
2. one channel (`903e6a20-0dea-42b1-8dd5-86afbec496ac`, "Classics", `channel-classics`)
3. two channels (pins first-channel-as-fallback selection and block ordering)
4. special-character channel name (`Channel "Special" \ Test` — pins `escLiq`)

The generator interpolates no render-time env (the only non-row input is the `SNC_TV_BROADCAST` constant), so snapshots are environment-independent. Commit the generated `.liq` snapshot files — they are the contract.

**Implementation notes:** capture snapshots from the CURRENT string-builder on a clean tree (run once to write, re-run to verify stable). Keep the existing `toContain` tests — they document intent; goldens document totality.

**Acceptance criteria:**
- [ ] Build + API unit suite pass
- [ ] Four committed `.liq` snapshot files, each generated by the unmodified current generator
- [ ] Re-running the suite is snapshot-stable (no obsolete/written warnings)

**Risk:** Low — test-only addition.
**Rollback:** revert the commit; no production code touched.
