# Player cold-start skeleton + HLS error state

Implements **Unit 1** of the parent feature's design (read `## Implementation Units`
→ Unit 1 in the parent body for exact signatures, CSS, and notes).

## Scope

`apps/web/src/components/media/global-player.tsx` +
`apps/web/src/components/media/global-player.module.css` only. Adds a local
`PlayerStatus` (`loading | ready | error`) driven by Vidstack's `onCanPlay`/`onError`
callback props, a pulsing 16:9 skeleton overlay during the loading window (covers both
the dynamic-module load and the HLS handshake — the audit's 12–15s dead air), a custom
error overlay with honest copy and a "Try again" remount (key bump), and a
`pendingFrame` class so the expanded container reserves 16:9 height before Vidstack
mounts. Non-audio media only. `pulse` keyframes move here from live.module.css (the
orphan deletion in live.module.css belongs to the sibling `offline-loading` story).

**Spike first**: typecheck `onCanPlay`/`onError` on `MediaPlayer` (verified in
@vidstack/react 1.12.13 types via `ReactElementProps<MediaPlayerInstance>`); fallback
is a `useMediaState` bridge child — see parent `## Risks`.

## Implementation notes

**Vidstack event-prop spike: direct props typechecked successfully.** The `onCanPlay` and
`onError` props are generated by `ReactEventCallbacks<InferComponentEvents<MediaPlayerInstance>>`
from `can-play` and `error` event keys in the Vidstack type hierarchy. No fallback
(`useMediaState` bridge child) was needed.

**Callback signatures**: `onCanPlay` receives `(detail: MediaCanPlayDetail, nativeEvent)` and
`onError` receives `(detail: MediaErrorDetail, nativeEvent)`. Both are passed as `() => setStatus(…)`
(arguments ignored) — TypeScript accepts this because callback parameters are covariant.

**React hooks ordering**: `useState` for `status`/`retryKey` and the `mediaId` reset `useEffect`
are all declared before the `if (!state.media) return null;` early return, satisfying React's
hooks rules.

**`clsx` import added**: `import { clsx } from "clsx/lite";` following the project idiom.

**Container `clsx` composition**: `clsx(containerClass, !isAudio && status !== "ready" && styles.pendingFrame)`
— `pendingFrame` activates for non-audio media while status is `loading` or `error`, reserving
16:9 height before Vidstack mounts and keeping the error overlay anchored correctly.

**Error overlay sits above the still-mounted MediaPlayer**: `position: absolute; inset: 0` on
`.playerError` — hls.js internal retries continue underneath.

**Tests**: 11 tests total (7 existing + 4 new). All pass. New cases cover: skeleton on mount /
dismissed by canPlay; error overlay replaces skeleton; "Try again" resets to loading; audio
suppresses all overlays.

**Build**: `bun run --filter @snc/web build` passes with no new errors or warnings.

**Sibling test failures**: Two pre-existing failures in `chat-panel.test.tsx` appeared in the
working tree from a concurrently running sibling story agent. These are not regressions from
this implementation — confirmed by verifying that before this story's changes, the test suite
passes 153 files cleanly.
