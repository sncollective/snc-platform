# Widen playout_queue to a source-polymorphic schema

Unit 1 of `unified-channel-model-creator-content-playable`. Lands first ‚Äî it's the regression gate
the other three units build on.

## Scope
Make `playout_queue` carry either a playout item or a creator content row, mirroring the
already-polymorphic `channel_content` (`playout-queue.schema.ts:24-27`).

**File**: `apps/api/src/db/schema/playout-queue.schema.ts` + a migration generated via
`drizzle-kit generate` ‚Äî NEVER hand-write the SQL/journal/snapshot (`.claude/rules/drizzle-migrations.md`).

```ts
playoutItemId: text("playout_item_id").references(() => playoutItems.id, { onDelete: "cascade" }), // now nullable
contentId: text("content_id").references(() => content.id, { onDelete: "cascade" }),               // new, nullable
// table extras:
check("playout_queue_one_source", sql`num_nonnulls(playout_item_id, content_id) = 1`),
```

Workflow: edit schema ‚Üí `bun run --filter @snc/api db:generate` ‚Üí `bun run --filter @snc/api db:migrate`.

## Acceptance
- [x] Migration generated by drizzle-kit (`0032_awesome_richard_fisk.sql`; random two-word name, real ms-noise timestamp, snapshot present) and applied clean.
- [x] All pre-existing `playout_queue` rows satisfy the CHECK (170 rows, 0 with null `playout_item_id`) ‚Äî verified before and after migration.
- [x] A row with both source columns null, or both set, is rejected by the DB CHECK (Postgres `23514` in both cases).
- [x] Admin playout queue behavior unchanged at the schema level (admin rows set `playout_item_id`, `content_id` null ‚Üí `num_nonnulls = 1`). Read/transition code wiring is units 2‚Äì4.

## Notes
The URI-resolution layer (`resolveContentUri`, `playout-orchestrator.ts:88-112`) already resolves
from either source ‚Äî this story only unblocks the queue table to hold the content source.

## Implementation notes
- **Files changed**: `apps/api/src/db/schema/playout-queue.schema.ts` ‚Äî `playoutQueue` now
  mirrors `channelContent`: `playoutItemId` dropped `.notNull()` (nullable FK), new nullable
  `contentId` FK to `content.id` (ON DELETE cascade), plus a table-level
  `check("playout_queue_one_source", sql\`num_nonnulls(playout_item_id, content_id) = 1\`)`.
  Added imports: `sql` from `drizzle-orm`, `check` from `drizzle-orm/pg-core` (`content` was
  already imported).
- **Migration**: `apps/api/drizzle/migrations/0032_awesome_richard_fisk.sql` ‚Äî generated via
  `bun run --filter @snc/api db:generate` (random two-word name; real ms-noise timestamp
  `1782345098924` in `_journal.json` idx 32; snapshot `meta/0032_snapshot.json` present ‚Äî NOT
  hand-written). Statements: `ALTER COLUMN playout_item_id DROP NOT NULL`, `ADD COLUMN content_id`,
  `ADD CONSTRAINT` for the content FK, `ADD CONSTRAINT playout_queue_one_source CHECK (...)`.
- **Applied clean**: `bun run --filter @snc/api db:migrate` ‚Üí "migrations applied successfully".
  Pre-migration the dev DB held 170 `playout_queue` rows, **0** with `playout_item_id IS NULL`,
  so every existing row satisfies `num_nonnulls = 1` after the new `content_id` column lands NULL.
  ADD CONSTRAINT did not fail.
- **CHECK verified against the live DB** (post-migration):
  - column `content_id` exists and is nullable; `playout_item_id` is now nullable.
  - constraint def: `CHECK ((num_nonnulls(playout_item_id, content_id) = 1))`.
  - both-null insert ‚Üí **REJECTED** (Postgres `23514` check_violation).
  - both-set insert ‚Üí **REJECTED** (`23514`).
  - all existing rows satisfy the check; 0 violations.
- **Discrepancies from design**: none. Schema mirrors `channelContent` exactly as specified.
- **Typecheck ripple (expected ‚Äî deferred to later stories)**: `bun run --filter @snc/api typecheck`
  now fails **only** in `src/services/playout-orchestrator.ts` (lines 56, 213, 214, 292, 293, 338):
  `playoutItemId` is now `string | null` in `$inferSelect` and the new `contentId` column appears
  in the row type, which the orchestrator's mapping/filter code does not yet handle. Per the story
  chain this is the regression-gate widening ‚Äî fixed by the transitions/reads stories (units 2‚Äì4),
  NOT this one. Left unpatched intentionally; no unexpected breakage anywhere else.
- **Adjacent issues parked**: none.
