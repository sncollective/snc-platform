# Low-Latency WebRTC Mode (WHIP/WHEP)

Upgrade the live stream delivery path from HLS to WebRTC using SRS WHIP (ingest) and WHEP (playback) for sub-second latency. Enables interactive features that require tight viewer-broadcaster synchrony (live Q&A, real-time viewer reactions visible to the broadcaster). Vidstack supports WHEP natively. Current HLS latency (~5-15s) is acceptable for most content; WebRTC is a future upgrade when interactive latency becomes a product priority.
