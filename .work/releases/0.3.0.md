# Release 0.3.0 — Animal Future 2026-04-24 Readiness

First public-event release. Ships everything needed for the Animal Future IRL streaming show on 2026-04-24 — the collapsed `0.2.x` post-streaming consolidation (admin polish, playout channel architecture, identity, access model, community + moderation groundwork, streaming account connect, notifications, email harness) together with the event-enabling scope (landing-page redesign with event row, mobile-nav + responsive overhaul, design-system foundation + adoption, chat/presence/reactions, notification inbox, live-page UX polish, uploads hardening, patron badges).

Originally scoped across `release-0.2.1` through `release-0.2.9` and `release-0.3.0` legacy boards; migrated into `platform/.memory/active/` during `boards-migration` (2026-04-18), then consolidated under a single 0.3.0 bundle (2026-04-20) once the April 24 event emerged as the real delivery target.

- **0.2.x consolidation (already code-done):** admin polish, playout channel architecture phases 1–3, playout retry + metadata autofill + review fixes, creator-lifecycle + team/access model, invites, notifications + community-notifications, streaming-account-connect, social-login-identity, mailpit dev SMTP harness, multipart upload fix, performance fixes, design-system-foundation (ark-ui, lucide, tokens), seed/demo + integration test scaffolding.

- **Deferrable stragglers:** systemd-graceful-exit, failed-upload-blocks-retry, hdr-video-tone-mapping, playout-queue-cleanup — split to 0.3.1 if they don't clear review before gate time.
- **Gate-surfaced additions (2026-04-24):** 5 refactor-gate blockers + 5 security-gate blockers + 2 testing-gate tests + 1 docs-gate update, all fixed fix-in-flight during `/release-deploy`. See Quality gate posture below.

## Deploy runbook

Deployment is user-station work (see root `scripts/platform/` and `platform/deploy/` for the prod deploy workflow). The release-deploy skill marks `status: released` and archives items; actual prod push happens outside this skill's scope.

Prod-side dependencies before deploy:
- `CORS_ORIGIN` set to the web frontend origin (validates the `getFrontendBaseUrl` fix lands correctly for invite emails + redirects).
- `BETTER_AUTH_URL` set to the API hostname (per existing config convention).
- SMTP relay configured (mailpit is dev-only).
- S3 credentials for Garage + tusd; `tusd` container running.
- OAuth credentials for Twitch / YouTube / Google / Apple / Mastodon as applicable.
